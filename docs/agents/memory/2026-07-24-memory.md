## Today's Question

What forms of memory help agents over long interactions, and what system-level architectures exist for agent-native memory?

## Main Paper

### Metadata

- Title: Are We Ready For An Agent-Native Memory System?
- Authors: Wei Zhou, Xuanhe Zhou, Shaokun Han, Hongming Xu, Guoliang Li, Zhiyu Li, Feiyu Xiong, Fan Wu
- Year: 2026
- Venue: arXiv (2606.24775)
- Link: https://arxiv.org/abs/2606.24775

### Why this paper?

This is the ideal Day 1 paper for the Memory topic. It provides a **systematic framework** that decomposes agent memory into four core modules, evaluates 12 representative systems across 11 datasets, and identifies open architectural trade-offs. Instead of listing random memory tricks, it builds a coherent map of the entire design space — exactly what the topic question asks for.

### Core Problem

Agent memory has evolved from simple retrieval-augmented mechanisms into sophisticated data management systems. But the field lacks a shared analytical framework to compare designs. Existing evaluations use only end-to-end task metrics (F1, BLEU), treating the whole system as a black box. Critical concerns like operational costs, architectural trade-offs across modules, and robustness under dynamic updates remain unexamined.

### Main Idea

The paper proposes decomposing agent memory into four core modules:

1. **Memory Representation and Storage** — How is information structured? Flat key-value stores, structured topic documents, graph-based relational memory, compressed summaries.

2. **Extraction** — How is raw experience transformed into storable representations? Summarization, fact extraction, event segmentation, or raw verbatim storage.

3. **Retrieval and Routing** — How is relevant information found at query time? Semantic search, keyword match, agentic multi-step retrieval, hierarchical routing to specialized stores.

4. **Maintenance** — How is memory kept up to date over time? Fact revision, consolidation, pruning, conflict resolution.

They evaluate 12 memory systems (MemGPT, MemoChat, MemoryBank, Infini Memory, Generative Agents' memory stream, ChatDev's message memory, etc.) plus two baselines across five workload types (fact-qa, preference tracking, conversation history, multi-session planning, knowledge updating).

### Key Findings

- **No single architecture dominates** — effectiveness depends on alignment between memory structure and workload bottleneck.
- **Representation matters most for fact-qa**: structured topic documents beat flat records.
- **Retrieval precision degrades under update**: systems with append-only storage fail when facts need revision.
- **Agentic retrieval (iterative LLM-guided search) helps** on complex queries but adds latency.
- **Localized maintenance** (targeted fact updates) is more cost-efficient than global reorganization.
- **Cost-performance trade-offs** are significant: more sophisticated memory reduces per-query accuracy but increases storage and retrieval time.

### Limitations

- The benchmark workloads are synthetic and may not capture real-world agent interaction dynamics.
- Only includes open-source/accessible systems — commercial agent memory (e.g., ChatGPT's internal mechanisms) are excluded.
- The framework evaluates each module in isolation through ablation, but real agents exhibit complex cross-module interactions.
- Does not cover memory **safety** or **privacy** concerns (addressed by other papers like MRMMIA).
- Latency measurements may not generalize to production deployments with concurrent multi-agent scenarios.

### Modern Perspective

This paper is a crucial bridge between the "agent memory as a feature" view (just add RAG) and the "agent memory as a first-class systems component" view. It shows that memory is not a single thing — it's a stack of design decisions with measurable trade-offs. The paper's framework is directly useful: it gives us a vocabulary to classify any new memory paper by its approach to representation, extraction, retrieval, and maintenance.

## Related Papers

### Paper 1: Position — Episodic Memory is the Missing Piece for Long-Term LLM Agents

- **Authors:** Mathis Pink, Qinyuan Wu, Vy Ai Vo, Javier Turek, Jianing Mu, Alexander Huth, Mariya Toneva (2025)
- **Link:** https://arxiv.org/abs/2502.06975

**Contributions:**
- Argues that current memory systems for LLM agents focus on **semantic memory** (general facts) while neglecting **episodic memory** (instance-specific, context-rich experiences).
- Identifies five key properties of biological episodic memory that should guide agent memory design: (1) single-shot encoding, (2) context-sensitive retrieval, (3) temporal organization, (4) constructive recombination (memories can be combined to support inference), (5) flexible generalization vs. specificity.
- Outlines a roadmap to integrate all five properties into future agent memory systems.

**Relation to main paper:** The main paper provides a general framework; this position paper argues for a **specific design philosophy** (episodic-first). Together they frame the central debate: should agent memory aim for general-purpose data management (main paper's implicit view) or specialization inspired by biological memory systems?

**Why it matters:** If episodic memory turns out to be the right abstraction, it would reshape the representation and retrieval modules of the main paper's framework. This is a thesis-relevant question.

### Paper 2: Infini Memory — Maintainable Topic Documents for Long-Term LLM Agent Memory

- **Authors:** Suozhao Ji, Baodong Wu, Zehao Wang, Lei Xia, Qingping Li, Ruisong Wang, Wenbo Ding, Zhenhua Zhu, Boxun Li, Guohao Dai, Yu Wang (2026)
- **Link:** https://arxiv.org/abs/2606.10677

**Contributions:**
- Proposes a **topic-structured document** architecture where each topic is a semantic unit with metadata, evidence collection, and revision history.
- Two-stage pipeline: (1) staging buffer collects raw observations, (2) periodic consolidation merges them into coherent topic documents.
- **Agentic retrieval**: LLM reads memory through iterative tool calls rather than a single retrieval step.
- Achieves 64.7% on MemoryAgentBench; ablations confirm topic-structure and iterative inspection improve complementary aspects.

**Relation to main paper:** Infini Memory is one of the 12 systems evaluated in the main paper, specifically implementing structured representation (topic documents) + agentic retrieval. Its 64.7% score on MemoryAgentBench provides a concrete benchmark anchor.

**Why it matters:** The topic-structured approach is a pragmatic compromise between unstructured flat records (simple but brittle) and fully normalized databases (powerful but complex). Worth deep-reading later as a design reference.

## Current Understanding

Agent memory is not a monolithic "add RAG and call it done" problem. It decomposes into at least four system-level design decisions: how to represent information, how to extract it from experience, how to retrieve it later, and how to maintain it over time. Early work (MemGPT, Generative Agents) used simple append-only streams or summary-based compression. The field is now converging on a more structured view:

- **Structured representation** (topic documents, graphs, typed records) beats flat storage for most workloads.
- **Agentic retrieval** (multi-step, LLM-guided) improves precision at the cost of latency.
- **Maintenance** is an underappreciated challenge — append-only systems degrade under dynamic facts.
- **Episodic vs. semantic memory** is a key design axis borrowed from cognitive science; which one matters more depends on the agent's task domain.

The core open problem: **there is no single best architecture**. The right memory design depends on workload characteristics (fact precision vs. contextual narrative, static vs. dynamic facts, single-session vs. cross-session tasks).

## Key Concepts

- Memory module decomposition: representation/storage, extraction, retrieval/routing, maintenance
- Semantic vs. episodic memory for agents
- Agentic (iterative LLM-guided) retrieval
- Topic-structured document memory
- Staging buffer + periodic consolidation
- Append-only vs. mutable memory
- Localized maintenance vs. global reorganization
- Workload-memory alignment principle

## Open Questions

1. **Representation choice.** What is the optimal granularity for memory representation — raw records, summarized facts, topic documents, or knowledge graphs? Does it depend on the agent's task domain in a predictable way?
2. **Episodic vs. semantic.** Should agent memory prioritize semantic memory (general facts, reusable procedures) or episodic memory (instance-specific experiences)? Can a single system support both?
3. **Maintenance under real conditions.** How do existing memory systems handle contradictory facts, outdated information, and conflicting evidence in real deployments?
4. **Cost-quality frontier.** What is the optimal trade-off between retrieval quality and latency/cost for different agent workloads?
5. **Memory as a security boundary.** How do we prevent memory poisoning (adversarial content injected into memory that later influences critical decisions)?
6. **Scaling memory.** How do memory systems perform with thousands of sessions, petabytes of stored experiences?
7. **Cross-agent memory.** Can memory systems be shared across multiple agents working on related tasks, and what sharing granularity is safe?
8. **Evaluation beyond task success.** What metrics capture memory quality directly — retrieval precision, fact update correctness, memory evolution quality?

## Possible Thesis Ideas

- **Workload-aware memory architecture selection** — Given a task description, predict which memory representation (flat record, topic document, graph) and retrieval strategy (direct, agentic) will yield the best cost-quality trade-off. The main paper's framework makes this tractable: it defines the design space; a learned selector could navigate it.

- **Unified episodic-semantic memory for agents** — Design a single memory system with two tiers: episodic (raw, context-rich, temporally indexed) for experience replay and semantic (compressed, relational, reusable) for general knowledge. The key challenge is how to consolidate episodic traces into semantic structures automatically.

- **Self-maintaining memory through LLM-driven consolidation** — An autonomous memory agent that detects contradictions, identifies stale facts, and reorganizes its representation without human intervention. The maintenance module is currently the least explored; this could be a first-principles design.
