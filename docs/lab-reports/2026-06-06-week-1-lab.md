## Cross-Cutting Themes

### 1. Sampling / Inference Efficiency is Everywhere

Every course has a "fast inference" variant that makes the technology practical:

| Course | Full method | Fast variant | Speedup |
|--------|-------------|--------------|---------|
| Diffusion | DDPM (1000 steps) | DDIM (10-50 steps) | 10-50× |
| LLM Serving | Naive batching | Continuous batching | 10-36× |
| Agents | Full ReAct trajectory | Toolformer (fine-tuned) | Variable |

**Insight:** Practical deployment always requires an order-of-magnitude speedup over the research prototype. This pattern is so universal it deserves a name — maybe "practicality gap."

### 2. Contrastive Learning as a Universal Alignment Mechanism

CLIP uses contrastive learning for vision-language alignment. This parallels:
- The diffusion training objective (contrastive-like: distinguish true noise from predicted noise)
- The agent's action selection (contrast between good/bad actions via observation feedback)

**Insight:** Contrastive objectives may be a universal mechanism for aligning different modalities (vision-language, noise-data, action-observation).

### 3. The ReAct Loop Appears Everywhere

The Thought → Action → Observation pattern is not just for agents:
- Diffusion reverse process: predict noise → denoise → observe result → repeat
- LLM inference: token prediction → KV cache update → attend → repeat

**Insight:** Any iterative refinement process maps to the ReAct pattern. This suggests a unifying framework where "agent" and "generative model" are different instances of the same iterative refinement class.

### 4. Memory is the Bottleneck

- Agents: context window limits trajectory length (ReAct bottleneck)
- LLM Serving: KV cache dominates GPU memory (PagedAttention addresses this)
- Diffusion: no explicit memory, but denoising steps serve as a form of working memory
- VLM: CLIP dual-encoder avoids cross-modal attention, but connector training is memory-intensive

**Insight:** Memory (what to keep, how to compress it, how to access it efficiently) is the fundamental bottleneck across all four courses. Each field solves it differently: agents use context window, serving uses PagedAttention, diffusion uses iterative refinement.

## Thesis Directions — Prioritized

### Priority 1: Adaptive ReAct Depth ⭐

**Problem:** The Thought-Action-Observation loop wastes tokens when the agent either overthinks (too many reasoning steps for a simple tool call) or underthinks (jumps to action without sufficient planning).

**Idea:** Dynamically decide between Thought and Action at each step based on uncertainty estimation from the LLM's token probabilities. High uncertainty → more reasoning; low uncertainty → act.

**Connection to this week:** Directly extends ReAct (Day 2 agents). Also relates to DDPM's variable step count (Day 1 generative) — both face the same quality-speed tradeoff.

### Priority 2: Unified Contrastive-Generative VLM Pretraining

**Problem:** CLIP gives good representations; generative finetuning (LLaVA, etc.) gives good generation. They use different losses on different data. Could a single loss do both?

**Idea:** Combine contrastive alignment with masked image-text modeling in a single pretraining stage, eliminating the two-stage paradigm.

**Connection to this week:** Directly extends CLIP (Day 2 multimodal). Also informed by diffusion's denoising objective (Day 3 generative) which is itself a form of masked reconstruction.

### Priority 3: Adaptive KV Cache for Agent Workloads

**Problem:** Agent workloads have unpredictable context patterns — bursty tool calls, long observations, repeated planning cycles. PagedAttention assumes uniform attention patterns.

**Idea:** A KV cache allocator that predicts which pages will be reused (repeated observations, system prompts) and which will be discarded (stale reasoning).

**Connection to this week:** Combines LLM serving (Day 1 llm-systems) with agent architecture (Day 1+2 agents). Cross-course ideas are rare and valuable.
